## 빈(Bean) 이란?
- 스프링 IoC 컨테이너가 관리하는 객체
> **스프링 IoC 컨테이너**
> - `ApplicationContext`가 Spring IoC Container를 대표하고, 빈을 인스턴스화하고 구성 및 모으는 책임이 있다.

### 빈 등록
1. 어노테이션 기반
2. Java 설정 클래스 기반
3. xml 기반

### 스프링 IoC 컨테이너는 어떻게 빈을 관리할까?
- 객체 생성 + property 설정 -> 의존 설정 -> 초기화 -> 사용 -> 소멸

## @Component Vs @Bean
| 구분 | @Bean | @Component |
| :--- | :--- | :--- |
| **적용 대상** | 메소드에 사용 | 클래스에 사용 |
| **주요 용도** | 개발자가 컨트롤이 불가능한 외부 라이브러리 사용 시 사용 | 개발자가 직접 컨트롤이 가능한 내부 클래스에 사용 |
